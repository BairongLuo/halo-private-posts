# 路线与进度

本文档记录 `halo-private-posts` 从“独立私密文章验证”收口到“Halo 原生文章增强能力”的推进状态。

## 当前产品方向

目标不是单独再做一套文章系统，而是把私密正文做成 Halo 原生 `Post` 的扩展能力：

- 文章仍然是 Halo 原生 `Post`
- 正文密文通过注解和 `PrivatePost` 存储
- 阅读体验优先收口到原文章页
- 作者恢复能力默认由平台恢复补上

## 已完成阶段

### 阶段 0：基础链路验证

- `PrivatePost` 扩展资源建模
- 匿名 bundle 查询与独立阅读页
- 浏览器本地密码解密
- 自动重锁

### 阶段 1：Halo 原生文章流接入

- 文章列表中的私密正文状态字段
- 文章编辑页顶部与 `Settings` 同级的“文章加密”入口
- 文章注解到 `PrivatePost` 的自动同步
- 原文章页正文区域内联锁定态
- 独立阅读页保留为独立入口

### 阶段 2：平台恢复主链路

- `EncryptedPrivatePostBundle v3`
- `password_slot`
- `site_recovery_slot`
- 站点恢复密钥生成与持久化
- 通过平台恢复后台重置访问口令

## 当前状态

主链路已经跑通：

- 作者可以在文章编辑页顶部的“文章加密”入口里直接加锁
- 再次加锁前会清理软删除残留，避免设置成功但列表和实际未生效
- 取消加锁后会先移除注解，再补删 `PrivatePost` 镜像
- 新文章会写入 `v3` bundle
- 原文章页和独立阅读页都可以密码解锁
- 带 `site_recovery_slot` 的文章可以由后台直接重置访问口令
- 平台恢复私钥只保存在服务端
- 匿名阅读页和 bundle JSON 接口都已关闭缓存，避免读到旧 bundle

## 仍需继续打磨的点

- 锁定态与正文态和不同主题的视觉融合
- 更大范围的 Halo 版本和主题兼容性回归
- 真实 Halo 环境下的完整发布演练与问题归档

## 当前发布判断

当前更适合视为：

- 可继续灰度部署
- 已具备基础 CI 与发版资产出口
- 公开市场发布前仍建议补足更多真实主题和版本组合回归

## 下一阶段计划

### 阶段 3：发布前收尾

- 继续削减误导性的旧文案和旧流程入口
- 让主题变量和版式适配更自然
- 跑完整 `安装 -> 加锁 -> 阅读 -> 覆盖口令 -> 删除插件` 回归
- 收紧卸载清理失败时的可见性和恢复策略
- 扩大真实 Halo 版本与主题组合的 smoke 回归

### 阶段 4：发布与运维

- 应用市场发布资产
- 基于 tag 的稳定版本发布节奏
- 更稳定的版本发布流程

## 暂不计划扩展的方向

- 多作者协作下的复杂密钥撤销
- 与会员系统、支付系统深度耦合

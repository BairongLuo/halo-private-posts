# 路线与进度

本文档用于记录 `halo-private-posts` 从“协议适配验证”走向“高集成 Halo 私密正文体验”的推进状态。

## 当前产品方向

目标不是再做一套独立文章系统，而是把“私密正文”做成 Halo 原生文章的一种增强能力：

- 文章仍然是 Halo 原生 `Post`
- 插件负责保存密文正文映射
- 文章继续出现在正常文章列表、正常主题和正常路由里
- 读者阅读时看到的是原文章体验上的锁定态和本地解锁

## 已完成

### 阶段 0：协议与独立阅读页验证

- `PrivatePost` 扩展资源建模完成
- `EncryptedPrivatePostBundle v1` 消费链路完成
- reader 本地解密与自动重锁完成
- 使用 `ZKVault` fixture 的前端兼容性测试完成

### 阶段 1：开始贴近 Halo 原生文章流

- 在 Halo 文章列表增加“私密正文”状态字段
- 文章列表可直接跳到对应文章的私密正文配置页
- 管理页支持按 `postName` 载入现有 Halo 文章
- 管理页展示文章摘要、slug、发布时间和可见性
- 保存或删除映射后同步刷新文章列表中的状态
- reader 样式改为 CSS 变量驱动，允许主题覆盖背景、文字、强调色和字体

## 当前仍存在的问题

- 私密正文配置仍然是独立管理页，不在 Halo 原生文章编辑器里
- 作者仍需要自己准备 bundle JSON，作者流不够自然
- 阅读页虽然开始支持主题变量，但仍是独立阅读路由，不是原文章页内原位解锁
- 主题接入仍需要模板层做额外接线

## 下一阶段计划

### 阶段 2：编辑流集成

优先级最高：

- 在 Halo 文章编辑流中提供私密正文配置入口
- 不再要求作者手工粘贴 bundle JSON
- 支持“输入正文 + 密码 -> 浏览器本地生成 bundle -> 保存映射”
- 在文章列表中持续显示私密正文状态

### 阶段 3：阅读体验高集成

- 在原文章页里直接渲染锁定态
- 解锁后在原页面正文区域展示内容
- 继续收敛 reader 样式，让主题布局、暗色模式和排版规则自然接管
- 将独立 `/private-posts?slug=...` 阅读页降级为兼容入口，而不是主入口

### 阶段 4：发布与运维

- 完整的 JDK 21 构建与安装验证
- 应用市场发布资产
- CI 构建与测试流水线
- 更完整的安装、主题接入和安全说明

## 当前建议

短期目标应优先把阶段 2 做完。只有当私密正文进入正常文章编辑流之后，这个插件才会真正具备“普通站长可用”的基础。
